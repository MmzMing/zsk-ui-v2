# 登录功能 - Cloudflare Turnstile 人机验证集成指南

本文档详细说明如何在登录流程中集成 Cloudflare Turnstile 人机验证，以增强系统安全性，防止暴力破解和自动化攻击。

## 1. 总体流程

1.  **前端展示**：在登录页面（用户名/密码输入框下方）加载 Turnstile 验证组件。
2.  **用户交互**：用户点击验证（或自动验证），Cloudflare 返回一个临时的 `token`。
3.  **提交请求**：前端将 `token` 连同 `username` 和 `password` 一起发送给后端登录接口。
4.  **后端验证**：后端收到请求后，优先调用 Cloudflare API 验证 `token` 的有效性。
5.  **结果处理**：
    *   **验证通过**：后端继续执行正常的账号密码校验逻辑。
    *   **验证失败**：后端直接拒绝请求，返回错误信息（如“人机验证失败”）。

---

## 2. Cloudflare 申请与配置

在开发前，需要先在 Cloudflare 后台申请验证密钥。

1.  **注册/登录**：访问 [Cloudflare Dashboard](https://dash.cloudflare.com/) 并登录。
2.  **进入 Turnstile**：在左侧菜单或产品列表中找到 **Turnstile**。
3.  **添加站点 (Add Site)**：
    *   **Site Name**：填写项目名称（如 `ZSK-UI`）。
    *   **Domain**：填写允许使用的域名。
        *   **本地开发**：可以添加 `localhost` 或 `127.0.0.1`。
        *   **生产环境**：添加实际上线的域名（如 `example.com`）。
    *   **Widget Mode**：选择 **Managed**（推荐，根据风险自动选择交互或无感验证）。
4.  **获取密钥**：创建成功后，保存以下两个密钥：
    *   **Site Key (站点密钥)**：**公开**，用于前端组件配置。
    *   **Secret Key (密钥)**：**保密**，用于后端 API 验证，**切勿泄露给前端**。

## 3. 前端开发指南 (React)

### 3.1 安装依赖

推荐使用 `@marsidev/react-turnstile`，它提供了良好的 React 封装和 TypeScript 支持。

```bash
npm install @marsidev/react-turnstile
```

### 3.2 环境变量配置

在 `.env` 文件中添加 Site Key：

```env
# Cloudflare Turnstile Site Key
VITE_TURNSTILE_SITE_KEY=你的_SITE_KEY_在这里
```

### 3.3 登录页面改造

修改登录组件（如 `src/pages/Auth/Login/index.tsx`），引入 Turnstile 组件。

```typescript
import { useState } from 'react'
import { Turnstile } from '@marsidev/react-turnstile'

export default function LoginPage() {
  const [turnstileToken, setTurnstileToken] = useState('')
  
  // ... 其他状态 (username, password, loading)

  const handleLogin = async (e) => {
    e.preventDefault()
    
    // 1. 校验是否完成验证
    if (!turnstileToken) {
      alert('请完成人机验证')
      return
    }

    try {
      // 2. 发送请求携带 token
      await request.post('/auth/login', {
        username,
        password,
        turnstileToken // 新增字段
      })
      // ... 登录成功逻辑
    } catch (error) {
      // ... 错误处理
      // 建议：失败后重置验证码状态，让用户重新验证
      setTurnstileToken('') 
    }
  }

  return (
    <form onSubmit={handleLogin}>
      {/* 用户名密码输入框 ... */}
      
      {/* 人机验证组件 */}
      <div className="my-4 flex justify-center">
        <Turnstile 
          siteKey={import.meta.env.VITE_TURNSTILE_SITE_KEY}
          onSuccess={(token) => setTurnstileToken(token)}
          onError={() => setTurnstileToken('')}
          onExpire={() => setTurnstileToken('')}
        />
      </div>

      <button type="submit">登录</button>
    </form>
  )
}
```

## 4. 后端开发指南 (Java Spring Boot)

后端需要在处理登录逻辑**之前**，先向 Cloudflare 验证 Token。

*   **验证接口 URL**: `https://challenges.cloudflare.com/turnstile/v0/siteverify`
*   **请求方法**: `POST`
*   **Content-Type**: `application/x-www-form-urlencoded`

### 4.1 验证参数

| 参数名 | 必填 | 说明 |
| :--- | :--- | :--- |
| `secret` | 是 | **Secret Key** (从 Cloudflare 后台获取) |
| `response` | 是 | 前端传来的 `turnstileToken` |
| `remoteip` | 否 | 用户的 IP 地址 (推荐加上，增强安全性) |

### 4.2 配置文件 (application.yml)

在配置文件中添加 Cloudflare Turnstile 的相关配置。

```yaml
cloudflare:
  turnstile:
    # 务必替换为你在 Cloudflare 后台获取的 Secret Key
    secret-key: "1x0000000000000000000000000000000AA" 
    verify-url: "https://challenges.cloudflare.com/turnstile/v0/siteverify"
```

### 4.3 Service 实现

创建一个 Service 用于处理与 Cloudflare 的交互。

```java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.http.HttpEntity;
import org.springframework.http.HttpHeaders;
import org.springframework.http.MediaType;
import org.springframework.http.ResponseEntity;
import org.springframework.stereotype.Service;
import org.springframework.util.LinkedMultiValueMap;
import org.springframework.util.MultiValueMap;
import org.springframework.web.client.RestTemplate;

import java.util.Map;

@Service
public class TurnstileService {

    @Value("${cloudflare.turnstile.secret-key}")
    private String secretKey;

    @Value("${cloudflare.turnstile.verify-url}")
    private String verifyUrl;

    private final RestTemplate restTemplate;

    public TurnstileService(RestTemplate restTemplate) {
        this.restTemplate = restTemplate;
    }

    /**
     * 验证 Turnstile Token
     *
     * @param token    前端传来的 turnstileToken
     * @param remoteIp 用户 IP 地址 (可选)
     * @return true 验证通过, false 验证失败
     */
    public boolean verify(String token, String remoteIp) {
        if (token == null || token.isEmpty()) {
            return false;
        }

        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_FORM_URLENCODED);

        MultiValueMap<String, String> map = new LinkedMultiValueMap<>();
        map.add("secret", secretKey);
        map.add("response", token);
        if (remoteIp != null) {
            map.add("remoteip", remoteIp);
        }

        HttpEntity<MultiValueMap<String, String>> request = new HttpEntity<>(map, headers);

        try {
            ResponseEntity<Map> response = restTemplate.postForEntity(verifyUrl, request, Map.class);
            Map<String, Object> body = response.getBody();
            // Cloudflare 返回的 JSON 中 success 为 true 即表示通过
            return body != null && Boolean.TRUE.equals(body.get("success"));
        } catch (Exception e) {
            // 记录日志
            e.printStackTrace();
            return false;
        }
    }
}
```

### 4.4 Controller 使用

在登录接口中调用 Service 进行验证。

```java
import jakarta.servlet.http.HttpServletRequest;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/auth")
public class AuthController {

    private final TurnstileService turnstileService;
    // private final UserService userService; 

    public AuthController(TurnstileService turnstileService) {
        this.turnstileService = turnstileService;
    }

    @PostMapping("/login")
    public ResponseEntity<?> login(@RequestBody LoginRequest loginRequest, HttpServletRequest request) {
        // 1. 人机验证
        // 获取真实 IP，注意如果有 Nginx 代理可能需要从 X-Forwarded-For 头获取
        String clientIp = request.getRemoteAddr(); 
        boolean isHuman = turnstileService.verify(loginRequest.getTurnstileToken(), clientIp);
        
        if (!isHuman) {
            return ResponseEntity.status(HttpStatus.BAD_REQUEST).body("人机验证失败，请刷新重试");
        }

        // 2. 账号密码验证 (原有的登录逻辑)
        // User user = userService.login(loginRequest.getUsername(), loginRequest.getPassword());
        
        return ResponseEntity.ok("登录成功");
    }
}

    public void setTurnstileToken(String turnstileToken) { this.turnstileToken = turnstileToken; }
    // ...
}
```

## 6. 常见问题 (FAQ)

### 6.1 Error 110200 (400 Bad Request)

如果在开发环境遇到 `Error: 110200` 或者是 400 错误，通常是因为 **域名不匹配**。

**原因**：
你在 Cloudflare 后台创建 Site Key 时，配置的域名列表（Domain Whitelist）中没有包含当前运行的域名（如 `localhost` 或 `127.0.0.1`）。

**解决方法**：
1.  **不需要内网穿透**：Cloudflare Turnstile 不需要服务器公网 IP，本地开发即可使用。
2.  **修改配置**：
    *   登录 Cloudflare Dashboard。
    *   找到对应的 Turnstile Widget -> Settings。
    *   在 **Domains** 列表中，添加 `localhost` 和 `127.0.0.1`。
    *   保存后等待约 1-2 分钟生效。
3.  **使用测试 Key**：
    *   如果不想修改真实 Key 的配置，可以使用 Cloudflare 提供的专用测试 Site Key：
    *   **Always Pass (总是通过)**: `1x00000000000000000000AA`
    *   **Always Block (总是拒绝)**: `2x00000000000000000000AB`
    *   测试 Key 可以在任何域名（包括 localhost）下工作。
