# 魔法链接登录流程设计

> 本文档详细描述了基于 Magic Link（魔法链接）的无密码登录方案的完整流程设计

---

## 一、流程概述

魔法链接登录是一种无密码登录方案，通过向用户邮箱发送一次性登录链接，用户点击链接即可完成登录。该方案具有安全性高、用户体验好的特点。

## 二、详细流程步骤

### 步骤 1：前端输入邮箱

**页面位置**：登录页

**交互流程**：
1. 用户在登录表单中输入注册邮箱
2. 前端进行基本格式校验（正则匹配邮箱格式）
3. 校验通过后进入人机验证环节

**前端代码示例**：
```typescript
interface LoginForm {
  email: string
}

const handleEmailSubmit = async (form: LoginForm) => {
  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/
  if (!emailRegex.test(form.email)) {
    toast.error('请输入有效的邮箱地址')
    return
  }
  
  // 进入人机验证环节
  await startCaptchaVerification()
}
```

---

### 步骤 2：人机验证

**验证方式**：滑块验证 / 点选验证 / 行为验证

**验证流程**：
1. 前端调用人机验证组件
2. 用户完成验证操作
3. 验证组件返回验证 token
4. 前端携带 token 发起登录请求

**安全说明**：
- 防止恶意批量发送邮件
- 验证 token 有效期较短（建议 5 分钟）
- 验证失败后需要重新验证

---

### 步骤 3：前端发起请求

**请求接口**：`POST /api/auth/send-magic-link`

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| email | string | 是 | 用户邮箱 |
| captchaToken | string | 是 | 人机验证 token |

**请求示例**：
```typescript
const response = await request.post('/api/auth/send-magic-link', {
  email: 'user@example.com',
  captchaToken: 'abc123xyz'
})
```

**成功响应**：
```json
{
  "success": true,
  "message": "登录链接已发送至您的邮箱"
}
```

---

### 步骤 4：后端发送邮件

**处理流程**：
1. 后端验证请求参数有效性
2. 查询数据库确认用户存在
3. 生成一次性登录 token（Magic Link Token）
4. 构建登录链接：`${FRONTEND_URL}/auth/magic-link?token=${token}`
5. 通过邮件服务发送包含链接的邮件

**Token 设计**：
- 长度：32-64 位随机字符串
- 有效期：15-30 分钟
- 存储：关联用户 ID 和过期时间，存入数据库或 Redis
- 安全性：使用加密算法生成，防止伪造

**邮件内容模板**：
```
【知识库小破站】登录验证

您好，

您正在尝试登录知识库小破站。请点击下方链接完成登录：

[登录链接]

该链接有效期为15分钟，请尽快使用。

如果这不是您的操作，请忽略此邮件。

知识库小破站团队
```

---

### 步骤 5：用户点击邮件链接

**链接格式**：
```
https://your-domain.com/auth/magic-link?token=abc123xyz789...
```

**用户体验**：
1. 用户收到邮件后点击链接
2. 浏览器打开链接，进入后端处理接口
3. 后端验证 token 有效性

---

### 步骤 6：后端验证 Token

**验证流程**：
1. 后端从 URL 参数中提取 token
2. 在数据库/Redis 中查找对应的 token 记录
3. 验证 token 是否存在且未过期
4. 验证通过后获取关联的用户信息

**失败处理**：
- Token 不存在：返回无效链接提示
- Token 已过期：提示链接已过期，建议重新获取
- Token 已使用：提示链接已失效

---

### 步骤 7：后端写入登录态（Cookie）

**登录态存储**：
- 使用 HttpOnly Cookie 存储 Session ID 或 JWT
- 设置安全标记：`HttpOnly`, `Secure`, `SameSite=Strict`
- 设置合理的过期时间（如 7 天）

**Cookie 配置示例**：
```typescript
res.cookie('sessionId', sessionId, {
  httpOnly: true,
  secure: process.env.NODE_ENV === 'production',
  sameSite: 'strict',
  maxAge: 7 * 24 * 60 * 60 * 1000 // 7天
})
```

---

### 步骤 8：后端跳转到首页

**重定向响应**：
```typescript
res.redirect(302, '/')
```

**重定向策略**：
- 成功验证后直接重定向到首页
- 支持携带跳转来源参数（可选）

---

### 步骤 9：前端已登录

**前端检测登录状态**：
1. 页面加载时检查 Cookie 是否存在
2. 通过 `/api/auth/me` 接口验证登录有效性
3. 更新前端状态管理（如 Zustand）
4. 显示用户信息和已登录状态

**状态同步代码示例**：
```typescript
const useAuth = create((set) => ({
  user: null,
  isLoggedIn: false,
  checkAuth: async () => {
    try {
      const response = await request.get('/api/auth/me')
      if (response.success) {
        set({ user: response.data, isLoggedIn: true })
      }
    } catch {
      set({ user: null, isLoggedIn: false })
    }
  }
}))
```

## 三、关键设计要点

### 3.1 安全性设计

| 安全措施 | 说明 |
|----------|------|
| HTTPS | 全程使用 HTTPS，防止中间人攻击 |
| HttpOnly Cookie | 防止 XSS 攻击窃取登录态 |
| SameSite Cookie | 防止 CSRF 攻击 |
| Token 过期 | Magic Link Token 有效期较短（15-30分钟） |
| Token 一次性 | 使用后立即失效，防止重复使用 |
| 邮件域名验证 | 验证邮箱域名合法性，防止滥用 |

### 3.2 用户体验优化

1. **快速反馈**：发送请求后立即提示用户检查邮箱
2. **过期提示**：链接过期后引导用户重新获取
3. **防重复发送**：限制单位时间内发送次数
4. **移动端适配**：确保邮件链接在移动端正常打开

### 3.3 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 邮箱不存在 | 返回友好提示，不泄露用户存在性 |
| 邮件发送失败 | 提示用户稍后重试 |
| Token 验证失败 | 清晰提示错误原因 |
| 网络超时 | 前端显示加载状态，支持重试 |

## 五、总结

魔法链接登录方案的核心优势：

1. **安全性**：无需密码，避免密码泄露风险
2. **便捷性**：用户只需点击链接即可登录
3. **兼容性**：无需安装额外应用，支持所有邮箱客户端
4. **可扩展性**：可与多因素认证结合使用

**适用场景**：
- 对安全性要求较高的系统
- 需要简化登录流程的产品
- 移动端优先的应用
