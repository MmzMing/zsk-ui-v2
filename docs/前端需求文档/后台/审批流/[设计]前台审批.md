## 功能需求

### 1. 审核队列列表
- **功能描述：** 统一展示所有前台内容的审核队列，支持按目标类型和审核状态筛选
- **展示内容：**
  - 内容标题（评论类型显示评论内容摘要）
  - 目标类型（文档/视频/文档评论/视频评论）
  - 分类编码（评论类型不显示）
  - 上传者信息
  - 风险等级（低/中/高）
  - AI审核标记
  - 审核状态
  - 创建时间

### 2. 审核队列查询
- **功能描述：** 支持多条件筛选审核队列
- **查询条件：**
  - 目标类型筛选（文档/视频/文档评论/视频评论）
  - 审核状态筛选（待审核/通过/驳回）
  - 分页查询

### 3. 审核操作
- **功能描述：** 对内容进行审核操作
- **审核操作：**
  - 通过审核
  - 驳回审核（需填写审核意见和选择违规原因）
  - 批量通过审核
  - 批量驳回审核

### 4. 审核详情
- **功能描述：** 查看内容审核详情
- **详情内容：**
  - 内容基本信息
  - 审核类型（AI审核/人工审核）
  - 审核状态
  - 风险等级
  - 审核意见
  - 违规原因列表
  - 审核人信息
  - 审核时间
  - AI审核结果详情（JSON格式）

### 5. 审核日志
- **功能描述：** 查看审核操作历史记录
- **展示内容：**
  - 内容标题
  - 目标类型
  - 审核人
  - 审核时间
  - 审核结果
  - 审核意见
  - 风险等级
- **查询条件：**
  - 目标类型筛选
  - 分页查询

### 6. 违规原因管理
- **功能描述：** 根据目标类型获取对应的违规原因字典
- **展示内容：**
  - 违规项ID
  - 违规原因标签

## 页面布局设计

### 整体布局
```
Card > CardBody
  |- 顶部标题栏（图标 + 标题 + 刷新按钮）
  |- Tab切换栏（审核队列 | 审核日志）
  |- 审核队列面板：
  |   |- 查询工具栏（目标类型筛选 + 审核状态筛选 + 重置 + 批量操作按钮）
  |   |- 表格区域（HeroUI Table, selectionMode="multiple"）
  |   |- 底部分页
  |- 审核日志面板：
  |   |- 查询工具栏（目标类型筛选 + 重置）
  |   |- 表格区域
  |   |- 底部分页
```

### 弹窗设计
- **审核详情弹窗**：展示审核记录的完整信息
- **审核操作弹窗**：通过/驳回操作，驳回时需填写审核意见和选择违规原因

## 数据流设计

### 审核队列
- `getAuditQueue(targetType, auditStatus, pageNum, pageSize)` → 队列列表
- `submitAudit(targetType, targetId, auditStatus, auditMind, violationIds)` → 单条审核
- `submitBatchAudit(targetType, targetIds, auditStatus, auditMind, violationIds)` → 批量审核

### 审核详情
- `getAuditDetail(targetType, targetId)` → 审核详情

### 审核日志
- `getAuditLogs(targetType, pageNum, pageSize)` → 日志列表

### 违规原因
- `getViolationReasons(targetType)` → 违规原因列表

## 枚举说明

### targetType 审核目标类型
| 值 | 说明 |
|---|------|
| 1 | 文档 |
| 2 | 视频 |
| 3 | 文档评论 |
| 4 | 视频评论 |

### auditStatus 审核状态
| 值 | 说明 |
|---|------|
| 0 | 待审核 |
| 1 | 审核通过 |
| 2 | 审核驳回 |
| 3 | 已撤回 |

### riskLevel 风险等级
| 值 | 说明 |
|---|------|
| low | 低风险 |
| medium | 中风险 |
| high | 高风险 |

### status 审核状态值（队列接口返回）
| 值 | 说明 |
|---|------|
| pending | 待审核 |
| approved | 已通过 |
| rejected | 已驳回 |
| withdrawn | 已撤回 |
