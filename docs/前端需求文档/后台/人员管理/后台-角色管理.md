# 后台-角色管理

**文件位置：** `src/pages/Admin/Personnel/Role.tsx`  
**最后更新日期：** 2026-04-20

## 功能需求

### 0. 页面布局
- **布局方式：** 左右分栏布局
- **左侧区域：**
  - 角色列表展示（表格形式）
  - 顶部：查询工具栏（角色名称搜索、状态筛选、新增按钮）
  - 底部：本地分页控件
  - 宽度固定，约占整体宽度的 35%
- **右侧区域：**
  - 选中角色的详情展示，使用 Tab 切换三个面板，使用heroui的tab组件库
  - Tab 1：基本信息（角色详情查看/编辑）
  - Tab 2：权限配置（菜单权限树形展示与分配）
  - Tab 3：关联用户（角色下的用户列表管理）
  - 未选中角色时显示空状态提示
  - 宽度自适应，约占整体宽度的 65%

### 1. 角色列表
- **功能描述：** 展示系统角色列表，支持搜索、筛选、本地分页
- **数据来源：** `GET /api/system/role/list`
- **查询条件：**
  - 角色名称（模糊查询，对应参数 `roleName`）
  - 角色状态（0正常 1停用，对应参数 `status`）
- **展示字段：**
  - 角色名称（roleName）
  - 角色权限字符串（roleKey）
  - 排序号（roleSort）
  - 状态（status）
  - 备注（remark）
- **分页方式：** 前端本地分页（后端一次性返回全量数据）
- **行操作：**
  - 点击行选中角色，右侧展示该角色详情

### 2. 角色管理
- **功能描述：** 管理系统角色的增删改查
- **功能包括：**
  - 新增角色（`POST /api/system/role`）
  - 编辑角色信息（`PUT /api/system/role`）
  - 删除角色，支持批量删除（`DELETE /api/system/role/{ids}`）
  - 启用/禁用角色（通过编辑接口修改 status 字段）
- **角色编辑字段：**
  - 角色名称（roleName，必填）
  - 角色权限字符串（roleKey，必填）
  - 排序号（roleSort，非必填）
  - 状态（status，默认0正常）
  - 备注（remark，非必填）

### 3. 角色权限
- **功能描述：** 配置角色的菜单权限
- **数据来源：** `GET /api/system/role/{roleId}/menus`
- **功能包括：**
  - 查看角色已分配的菜单ID列表
  - 权限树形展示（菜单树，包含菜单和按钮节点）
  - 全量更新角色权限（`PUT /api/system/role/{roleId}/menus`，传入完整的 menuIds 列表）
  - 追加权限（`POST /api/system/role/{roleId}/menus`，传入新增的 menuIds）
  - 移除权限（`DELETE /api/system/role/{roleId}/menus`，传入要移除的 menuIds）
- **交互方式：**
  - 树形勾选方式分配权限，勾选变更时调用全量更新接口
  - 支持全选/取消全选
  - 支持父子联动（勾选父节点自动勾选所有子节点）

### 4. 角色用户
- **功能描述：** 管理角色下关联的用户
- **数据来源：** `GET /api/system/role/{roleId}/users`
- **功能包括：**
  - 查看角色关联的用户ID列表
  - 全量更新角色用户（`PUT /api/system/role/{roleId}/users`，传入完整的 userIds 列表）
  - 追加用户（`POST /api/system/role/{roleId}/users`，传入新增的 userIds）
  - 移除用户（`DELETE /api/system/role/{roleId}/users`，传入要移除的 userIds）
- **交互方式：**
  - 用户列表展示已关联的用户信息
  - 支持添加用户（从系统用户列表中选择）
  - 支持移除单个用户或批量移除
