# 后台-文档创建与编辑

**文件位置：** `src/pages/Admin/Document/CreateEdit.tsx`
**最后更新日期：** 2026-04-26

---

## 功能概述

本文档合并了原有的「文档编辑」和「文档上传」功能，统一为「文档创建与编辑」入口。用户进入该页面后，通过两个卡片选择操作：编辑已有文档或创建新文档。创建文档支持手动创建空白文档或导入 Markdown 文件两种方式。

---

## 页面结构

### 1. 入口页面（文档创建与编辑首页）

- **路径**：`/admin/document/create-edit`
- **布局**：页面中央展示两个操作卡片，水平排列（移动端垂直排列）
- **卡片内容**：
  - **左侧卡片：编辑已有文档**
    - 图标：编辑/文档图标
    - 标题：编辑已有文档
    - 描述：查看并编辑已创建的文档内容
    - 点击行为：跳转到文档列表页 `/admin/document/list`
  - **右侧卡片：创建新文档**
    - 图标：新建/添加图标
    - 标题：创建新文档
    - 描述：手动创建新文档或导入 Markdown 文件
    - 点击行为：弹出创建方式选择弹窗

#### 弹窗：选择创建方式

- **弹窗标题**：选择创建方式
- **布局**：弹窗内左右两个按钮卡片
  - **左侧按钮：手动创建**
    - 图标：编辑图标
    - 标题：手动创建
    - 描述：创建一个空白文档，手动编辑内容
    - 点击行为：关闭弹窗，跳转到编辑器页面，内容为空
  - **右侧按钮：导入 Markdown**
    - 图标：上传图标
    - 标题：导入 Markdown
    - 描述：上传 .md 或 .markdown 文件，自动解析内容
    - 点击行为：触发文件选择器，选择文件后跳转到编辑器页面并解析内容

---

### 2. 编辑器页面

- **路径**：`/admin/document/editor`
- **布局**：左右分栏布局（移动端上下分栏或切换显示）
  - **左侧：编辑区**
    - 使用 `MarkdownEditor` 组件（`src/components/ui/editor/MarkdownEditor.tsx`）
    - 支持 Markdown 语法高亮和实时编辑
    - 参考test，需要支持主题切换（默认/暗黑）
    - 支持代码块语法高亮（如 JSON、SQL、Python 等）
    - 手动创建时：编辑器内容为空，显示占位符
    - 导入 Markdown 时：解析文件内容并填充到编辑器
  - **右侧：实时预览区**
    - 实时渲染左侧编辑的 Markdown 内容
    - 支持代码高亮、表格、图片等 Markdown 元素渲染

#### 导入 Markdown 流程

1. 用户在弹窗选择「导入 Markdown」
2. 触发文件选择器（仅接受 `.md`、`.markdown` 后缀文件）
3. 选择文件后，显示加载动效（如旋转加载器或进度条）
4. 前端读取文件内容（UTF-8 编码）
5. 解析完成后，跳转到编辑器页面，内容填充到 `MarkdownEditor`
6. 右侧预览区实时显示解析后的内容

#### 加载动效

- **触发时机**：选择 Markdown 文件后、文件内容解析完成前
- **展示方式**：全屏遮罩或编辑器区域中央显示旋转加载器
- **提示文字**：正在解析文件...

---

### 3. 文档元信息编辑弹窗

- **触发时机**：用户在编辑器页面点击「提交」或「保存」按钮时
- **弹窗内容**：复用文档列表页的「编辑详情」弹窗
- **字段**：
  - 文档标题（noteName）
  - 文档封面（coverFileId）
  - 文档分类（broadCode、narrowCode）
  - 文档标签（noteTags）
  - 文档摘要（description）
  - 文档等级（noteGrade）
  - 适合人群（suitableUsers）
  - 文档状态（status：草稿/已发布）
  - 是否置顶（isPinned）
  - 是否推荐（isRecommended）
  - SEO 标题（seoTitle）
  - SEO 描述（seoDescription）
  - SEO 关键词（seoKeywords）

## 交互流程

### 流程 1：手动创建新文档

```
进入入口页面
  → 点击「创建新文档」卡片
    → 弹出「选择创建方式」弹窗
      → 点击「手动创建」
        → 跳转到编辑器页面（内容为空）
          → 用户编辑内容
            → 点击「提交」
              → 弹出「文档元信息编辑」弹窗
                → 填写元信息并确认
                  → 调用 POST /api/document/docNoteDtlAggregate/full 保存
```

### 流程 2：导入 Markdown 文件

```
进入入口页面
  → 点击「创建新文档」卡片
    → 弹出「选择创建方式」弹窗
      → 点击「导入 Markdown」
        → 触发文件选择器
          → 选择 .md 或 .markdown 文件
            → 显示加载动效「正在解析文件...」
              → 解析完成，跳转到编辑器页面（内容已填充）
                → 用户可继续编辑
                  → 点击「提交」
                    → 弹出「文档元信息编辑」弹窗
                      → 填写元信息并确认
                        → 调用 POST /api/document/docNoteDtlAggregate/full 保存
```

### 流程 3：编辑已有文档

```
进入入口页面
  → 点击「编辑已有文档」卡片
    → 跳转到文档列表页
      → 选择文档并点击编辑
        → 跳转到编辑器页面（内容回填）
          → 用户编辑内容
            → 点击「提交」
              → 弹出「文档元信息编辑」弹窗（数据回填）
                → 修改元信息并确认
                  → 调用 PUT /api/document/docNoteDtlAggregate/{id}/full 更新
```

## 相关接口

### 创建笔记（元信息 + 正文）

- **接口**：`POST /api/document/docNoteDtlAggregate/full`
- **用途**：手动创建或导入后，提交文档元信息和正文
- **请求体**：
  ```json
  {
    "docNote": {
      "userId": "1",
      "noteName": "文档标题",
      "noteTags": "tag1,tag2",
      "description": "文档摘要",
      "coverFileId": "1",
      "broadCode": "tech",
      "narrowCode": "java",
      "noteGrade": 1,
      "noteMode": 1,
      "suitableUsers": "初学者",
      "status": 3,
      "auditStatus": 0,
      "publishTime": "2026-04-26 10:30:00",
      "isPinned": 0,
      "isRecommended": 0,
      "seoTitle": "SEO标题",
      "seoDescription": "SEO描述",
      "seoKeywords": "keyword1,keyword2"
    },
    "content": "# 文档正文\n\nMarkdown 格式的正文内容..."
  }
  ```

### 全量更新笔记（元信息 + 正文）

- **接口**：`PUT /api/document/docNoteDtlAggregate/{id}/full`
- **用途**：编辑已有文档后更新
- **请求体**：与创建接口类似，需包含 `docNote.id`

### 获取笔记全量信息

- **接口**：`GET /api/document/docNoteDtlAggregate/{id}/full`
- **用途**：编辑已有文档时，回填编辑器内容和元信息

### 上传 MD 文件并保存（可选）

- **接口**：`POST /api/document/docNoteDtl/upload`
- **用途**：如需后端解析 Markdown 文件，可调用此接口
- **说明**：前端导入场景建议前端直接读取文件内容，如需后端处理可调用此接口

## 编辑器功能

基于 `MarkdownEditor` 组件（`src/components/ui/editor/MarkdownEditor.tsx`）：

- **文本格式化**：加粗、斜体、下划线、删除线
- **标题层级**：H1 ~ H6
- **列表**：有序列表、无序列表、任务列表
- **代码块**：行内代码、代码块（支持语法高亮）
- **表格**：插入和编辑表格
- **链接**：插入超链接
- **图片**：插入图片（支持上传）
- **引用**：块引用
- **分割线**：水平分割线
- **实时预览**：右侧实时渲染 Markdown 内容

---

## 状态管理

- **编辑器内容**：使用 React state 管理，实时同步到预览区
- **导入文件状态**：管理文件选择、解析中、解析完成等状态
- **元信息表单**：使用表单状态管理弹窗内的字段值

---

## 注意事项

1. **文件格式限制**：导入 Markdown 时，仅支持 `.md` 和 `.markdown` 后缀文件
2. **编码格式**：导入文件需使用 UTF-8 编码读取
3. **内容校验**：提交前需校验编辑器内容非空
4. **移动端适配**：编辑器在移动端需优化触摸体验，工具栏支持横向滚动
5. **加载状态**：导入文件解析期间必须显示加载动效，避免用户重复操作
