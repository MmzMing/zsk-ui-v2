# [设计]通知公告管理页面

## 需求概述

后台管理页面，用于管理系统通知公告。支持公告的增删改查、批量删除、多条件搜索，以及向 B站/邮箱/QQ群 推送公告功能。

## 页面结构

### 整体布局

工具栏 + 表格布局，使用 Card 容器包裹：
- 顶部工具栏：图标标题、推送按钮组、新增、刷新
- 搜索栏：标题搜索、状态筛选、重置、批量删除
- 中部表格：公告列表，支持多选
- 弹窗：新增/编辑公告

### 1. 标题栏

- **左侧**：Bell 图标 + 「通知公告」标题
- **右侧按钮组**：
  - 推送 B站（Send 图标）：选择公告后推送至 B站
  - 推送邮箱（Mail 图标）：选择公告后推送至邮箱
  - 推送 QQ群（MessageCircle 图标）：选择公告后推送至 QQ群
  - 新增公告（Plus 图标）：打开新增弹窗
  - 刷新（RefreshCw 图标）：重新加载公告列表

### 2. 搜索栏

- **标题搜索**：Input 输入框，支持回车搜索、清空按钮
- **状态筛选**：Select 下拉框（正常/关闭）
- **重置按钮**：清空所有搜索条件
- **批量删除**：选择公告后显示，显示选中数量

### 3. 公告表格

- **选择模式**：多选（selectionMode="multiple"）
- **列定义**：
  - 公告标题（字体加粗）
  - 公告类型（Chip 标签：通知/公告，颜色区分）
  - 状态（Chip 标签：正常-绿色/关闭-红色，dot 变体）
  - 创建者（hidden lg:table-cell）
  - 创建时间（hidden lg:table-cell，格式化显示）
  - 操作（编辑 + 删除按钮）
- **空状态**：使用 StatusState 组件显示空状态
- **加载状态**：使用 StatusState 组件显示加载中

### 4. 编辑弹窗（新增/编辑共用）

- **表单字段**：
  - 公告标题（必填，Input）
  - 公告类型（必填，Select：通知/公告）
  - 状态（Select：正常/关闭）
  - 公告内容（必填，Textarea，4-8 行）
- **按钮**：取消、创建/保存
- **校验**：标题和内容必填

## 技术方案

### 文件位置

```
src/pages/Admin/Monitor/Notice.tsx
```

### 依赖

- `@heroui/react`：Button, Input, Select, Table, Modal, Card, Chip, Tooltip, Textarea
- `lucide-react`：Trash2, Plus, Pencil, Search, RefreshCw, Bell, Mail, MessageCircle, Send
- `@/utils/toast`：消息提示
- `@/utils/format`：日期格式化
- `@/components/ui/StatusState`：状态组件
- `@/api/admin/notice`：公告 API
- `@/types/notice.types`：类型定义和选项配置

### 状态管理

- `noticeList`：公告列表数据
- `isLoading`：加载状态
- `selectedKeys`：选中的公告 ID 集合
- `queryParams`：查询参数（标题、状态）
- `searchKeyword`：搜索关键词
- `editMode`：弹窗模式（create/edit）
- `editingNotice`：当前编辑的公告数据

### API 接口

| 接口 | 用途 |
|------|------|
| `getNoticeList` | 获取公告列表 |
| `createNotice` | 创建公告 |
| `updateNotice` | 更新公告 |
| `deleteNotice` | 删除公告（支持批量，逗号分隔 ID） |

### 交互细节

- 推送按钮：未选择公告时提示「请先选择要推送的公告」
- 批量删除：未选择时提示「请选择要删除的公告」
- 搜索：支持回车触发，清空输入时自动重置搜索
- 表格行：悬停显示操作按钮（编辑/删除）
- 删除：单行删除直接执行，批量删除需先选择

## 类型定义

### 公告类型

- `noticeType`：'1' 通知 / '2' 公告
- `status`：'0' 正常 / '1' 关闭

### 数据模型

```typescript
interface SysNotice {
  id: string
  noticeTitle: string
  noticeType: NoticeType
  content: string
  status: NoticeStatus
  createBy?: string
  createTime?: string
}
```
