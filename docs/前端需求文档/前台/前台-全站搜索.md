# 前台-全站搜索（文档搜索）

> 关联 API：`docs/后端接口/document/[document]全局搜索-api接口文档.md`
> 关联接口：`GET /api/document/search/all`
> 入口位置：前台 **页头（FrontHeader）** 导航中将原"文档管理"项改名为"文档"（仅文案变更，不动其它），点击进入站内搜索页（站内跳转，不开新标签）。

---

## 一、设计目标

打造一个 **简洁、克制、动效拉满** 的站内全站搜索体验。所有交互均控制在站内（SPA 路由），通过路由 + URL 参数同步搜索状态，可分享、可前进后退。

### 核心原则
- **一屏直达**：搜索框居中、下方即时出结果，不跳页二次刷新。
- **站内跳转**：详情页通过 React Router 导航，不打开新窗口。
- **状态可分享**：搜索关键字、类型、排序、视图模式全部写入 URL Query。
- **动效有意义**：用动效引导视觉焦点（焦点变化、列表切换、骨架→数据切换），不滥用。
- **无障碍**：键盘可全控（`/` 唤起搜索框，`↑/↓` 选建议项，`Enter` 提交，`Esc` 退出）。

## 三、页面结构

```
┌──────────────────────────────────────────────────────────────────┐
│  [SearchHero]      ← 搜索框 + 实时建议 + 历史记录 + 热门关键词      │
├──────────────────────────────────────────────────────────────────┤
│  [AnimatedTabs]    ← 类型 Tabs（aceternity-ui Animated Tabs 风格）  │
│                      ※ 该 Tabs 控制下方"筛选+结果"整块面板的切换    │
├──────────────────────────────────────────────────────────────────┤
│  ┌──────────── TabPanel（与 Tab 一一对应，整块切换） ────────────┐ │
│  │ [FilterAccordion] ← 手风琴折叠筛选栏（排序 / 分类 / 视图模式） │ │
│  │ ────────────────────────────────────────────────────────── │ │
│  │ [ResultStream]    ← 卡片网格 / 单列表 + 关键字高亮            │ │
│  │                     + 虚拟列表（@tanstack/react-virtual）    │ │
│  │                     + IntersectionObserver 懒加载追加         │ │
│  └────────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────────┘
```

页面整体为 **垂直自上而下** 的单列结构。**FilterAccordion 与 ResultStream 是一体的 TabPanel**，由 AnimatedTabs 统一控制：切换 Tab 时，下方"筛选 + 结果"整块作为一个整体一起被替换/切换（共享同一过渡动效与 `AnimatePresence` 容器），三种类型（全部/视频/文档）各自维护自己的筛选状态与结果流。

### 1. SearchHero 区
- 搜索框：HeroUI `Input`，左侧 `Search` 图标（lucide），右侧 `X` 清除 + `回车提交` 提示。
- **聚焦展开**：聚焦时输入框宽度从 480px 平滑过渡到 720px（或全宽 max-w-3xl），下方面板淡入显示：
  - **搜索历史**（localStorage，最多 10 条，可单条删除/清空）
  - **热门关键词**（前端写死或预留 API，5~8 个 chip）
  - **实时建议**（输入 ≥1 字，300ms 防抖触发轻量预查询，最多 6 条；可分阶段实现）
- **快捷键**：`/` 全局聚焦；`Esc` 失焦关闭面板；`↑/↓` 选项；`Enter` 提交。

### 2. AnimatedTabs 类型切换条
- 采用 **Aceternity UI 的 Animated Tabs**（`motion.div` + `layoutId` 共享背景胶囊滑动）实现 `全部 / 视频 / 文档`。
- **AnimatedTabs 是下方 FilterAccordion + ResultStream 一体面板的控制器**：每个 Tab 对应一个独立 TabPanel（含独立的筛选状态与结果流），切换 Tab 即整块切换。
- 选中项通过 `layoutId="search-tab-pill"` 让背景在三项之间平滑滑动；非选中项 hover 时背景轻微浮现。
- 切换时使用 `AnimatePresence mode="wait"` 包住整块 TabPanel，做"淡出 → 淡入上滑"过渡（不仅是结果区，连同筛选栏一起切换）。
- 组件落点：`src/components/ui/aceternity/AnimatedTabs.tsx`（如已存在则复用，没有则新建并供其他页面复用）。

```tsx
// 关键样式示意（不写死颜色，用 HeroUI token）
<button className="relative px-4 py-2 text-sm">
  {isActive && (
    <motion.span
      layoutId="search-tab-pill"
      className="absolute inset-0 rounded-full bg-primary/15 dark:bg-primary/25"
      transition={{ type: 'spring', bounce: 0.2, duration: 0.5 }}
    />
  )}
  <span className="relative z-10">{tab.label}</span>
</button>
```

### 3. FilterAccordion 手风琴筛选栏
- **不使用 Select 下拉**，统一改为 HeroUI `Accordion`（手风琴折叠面板），位于 AnimatedTabs 之下、ResultStream 之上，**横跨整行**（与结果区同宽，不分栏）。
- 三个分组（自上而下垂直排列，每个分组可独立展开/收起）：
  | 分组 | 选项形式 | 选项内容 |
  | :--- | :--- | :--- |
  | 排序方式 | 单选 chip / radio chip | `默认 / 最新 / 最热` |
  | 内容分类 | 多选/单选 chip 列表 | 字典或预设：`tech / life / study / tool / other`（横向 chip 流） |
  | 视图模式 | 双图标 segmented | `卡片 / 列表` |
- 折叠展开使用 HeroUI Accordion 自带动画；分组内选项点击带 `whileTap` 缩放反馈，选中态用 `layoutId` 共享背景滑动。
- Accordion 顶部右侧（trigger 行）显示当前生效条件徽标（chip 形式：例如 `最热 · tech · 卡片`），便于折叠时一眼识别当前状态。
- 默认展开"排序方式"与"视图模式"，"内容分类"默认收起以节省纵向空间。
- 在 AnimatedTabs 与 FilterAccordion 之间显示"共 N 条结果，关键字「xxx」"的实时摘要。

### 4. ResultStream 结果区（虚拟列表 + 懒加载，无分页）
- **数据加载策略**：
  - 首屏请求 `pageNum=1, pageSize=20`。
  - 滚动接近底部（距离底部 < 600px）时由 `IntersectionObserver` 哨兵触发下一批（`pageNum++`），直到 `loaded >= total`。
  - 状态：`idle / loading / loadingMore / done / error`，避免重复触发。
- **虚拟列表**：使用 `@tanstack/react-virtual`，根据视图模式选择不同行高估算：
  - 列表视图：`estimateSize = 112`（90 缩略图 + padding）
  - 卡片视图：以"行"为单位虚拟化，按响应式列数（1/2/3/4）将 items 切块成行，行高估算 `~280`
- 切换关键字、类型、排序、分类时清空已加载列表并重置 `pageNum=1`。
- 视图切换（卡↔列表）共享 items，仅切换布局并重置虚拟列表 measurement。

#### 卡片视图（默认）
- 网格响应式列数 `1/2/3/4`，每行作为虚拟化单元。
- 卡片元素：缩略图（16:9，hover 微缩放 1.03）→ 类型徽标（左上角）→ 时长徽标（右下角，仅视频）→ 标题（2 行截断 + 高亮）→ 描述（2 行截断 + 高亮）→ 底栏：作者 · 浏览量 · 点赞 · 标签前 2。
- 进入动画：每行 `staggerChildren 30ms`。

#### 列表视图
- 单列横向：左 160×90 缩略图，中间标题+描述+标签，右侧统计。
- hover：浅灰底 + 左侧 3px 主题色指示条（高度从 0 → 100% 展开）。

#### 通用交互
- **关键字高亮**：标题/描述匹配片段使用 `<mark>` + 主题色背景（深浅主题适配）。
- **点击行为**：
  - `type=video` → `useNavigate('/video/:id')`
  - `type=document` → `useNavigate('/document/:id')`
  - 站内 SPA 跳转，不开新标签。
- **加载占位**：首屏 8 张卡片骨架 / 6 行行骨架；追加加载在底部显示 1 个 spinner + "加载更多…"。
- **空结果**：插画 + "未找到与「xxx」相关的内容" + 热门关键词 chip。
- **加载到底**：底部展示 "已经到底啦 ~"。
- **错误**：底部 "加载失败，点击重试" 按钮，不破坏已加载内容。

## 四、动效规范（Framer Motion + HeroUI 内置）

| 场景 | 动效 |
| :--- | :--- |
| 搜索框聚焦 | width 平滑过渡 240ms，下方建议面板 `opacity 0→1` + `y 8→0`，180ms |
| 类型 Tab 切换 | aceternity Animated Tabs 共享 `layoutId` 胶囊滑动；结果区 `AnimatePresence mode="wait"` 旧内容上移淡出（120ms），新内容下方淡入上滑（180ms） |
| 手风琴展开/收起 | HeroUI Accordion 内置 height/opacity 过渡；选项 chip 选中态由 `layoutId` 共享背景滑动 |
| 卡片入场 | `staggerChildren: 0.03`，每张 `opacity 0→1` + `y 12→0`，180ms |
| 视图切换（卡↔列表） | `motion.div layout` + `LayoutGroup` 让结果项平滑过渡（FLIP），并触发 virtualizer `measure()` |
| 卡片 hover | `scale 1→1.02`，缩略图内 `scale 1→1.03`，阴影从 `shadow-sm` → `shadow-lg`，120ms |
| 高亮命中 | 关键字 `<mark>` 底色 `bg-primary-100/60`，hover 卡片时该底色提亮 |
| 懒加载追加 | 新批次 items 首次进入视口时再触发 stagger 入场，已渲染项不重复动画 |
| 清除关键字 | 搜索框右侧 `X` 旋转 90° 后内容淡出 |

> 全局动画时长统一 120~240ms 区间，缓动 `cubic-bezier(0.22, 1, 0.36, 1)`（HeroUI 默认）。

## 五、状态管理与 URL 同步

- 使用 `useSearchParams()` 读写 query：`keyword / type / sort / category / view`（**无 page**，列表为流式追加，pageNum 仅作为内部状态）。
- 输入框输入有 **400ms 防抖**，防抖结束后写 URL，由 URL 变化触发"清空 + 重新加载首批"。
- 切类型/排序/分类/清除关键字 → 立即重置 `pageNum=1` 并清空已加载列表。
- 视图切换不重置数据，只切换布局并触发 virtualizer `measure()`。
- 历史记录：localStorage key `zsk_search_history`，最多 10 条，按提交时间倒序去重。

```ts
// 类型定义建议
type SearchType = 'all' | 'video' | 'document'

interface SearchItem {

  playCount: number | null
  readCount: number | null
  likeCount: number
```

## 八、可访问性 / 响应式

- 移动端：搜索框全宽，AnimatedTabs 横向滚动；FilterAccordion 默认全部收起，节省纵向空间。视图切换仍可用（卡片在窄屏自动单列）。
- 触摸目标 ≥ 44px。
- 全部交互可由键盘完成；搜索框 `aria-label="全站搜索"`，结果列表用 `role="list"`、`role="listitem"`，Tab 用 `role="tablist"`，Accordion 使用 HeroUI 内置 ARIA。
- 关键字高亮文案保持原读音（仅视觉强调）。

## 九、主题与样式

- 颜色一律使用 HeroUI 语义 token：`bg-content1 / border-default-200 / text-default-500 / text-primary` 等，确保深浅主题切换正常。
