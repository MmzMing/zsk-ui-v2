# 前台-文档详情

> **版本**: 2.0\
> **更新日期**: 2026-04-29\
> **说明**: 基于后端 `/api/document/docHomeNote/*` 接口体系，采用渐进式加载 + 左中右三栏布局的文档详情页。

***

## 一、页面架构总览

### 1.1 渐进式加载策略

```
页面进入 → 获取元信息(/detail/{id}) → 构建骨架屏 → 获取交互信息(/interaction/{id})
                                                              ↓
                                           用户滚动至评论区 → 懒加载评论(/comments/{noteId})
```

- **第一步**：调用 `GET /api/document/docHomeNote/detail/{id}` 获取文档元信息（标题、内容、封面、分类、标签等），立即渲染骨架。
- **第二步**：调用 `GET /api/document/docHomeNote/interaction/{id}` 获取交互数据（浏览量、点赞/收藏数、作者粉丝数、当前用户状态），填充交互区域。
- **第三步**：用户滚动至评论区附近时，触发 `GET /api/document/docHomeNote/comments/{noteId}` 懒加载评论列表。

### 1.2 布局结构（左-中-右三栏）

## 二、左侧：Tracing Beam 装饰

### 2.1 组件说明

使用 [Aceternity UI Tracing Beam](https://ui.aceternity.com/components/tracing-beam) 组件，在页面左侧提供滚动驱动的光束动画效果。

**SVG 路径造型**：直线为主，在顶部有一个类似 Git 分支图的转弯折线（水平→垂直），其余部分保持垂直直线延伸。整体简洁，不添加多余节点/波浪。

### 2.2 技术要点

- 通过 `npx shadcn@latest add https://ui.aceternity.com/registry/tracing-beam.json` 引入组件源码。
- 依赖：`motion`（Framer Motion）、`clsx`、`tailwind-merge`。
- 光束沿 SVG 路径随页面滚动动态延伸，滚动速度影响光束长度。
- 仅作为装饰元素，无交互功能。

### 2.3 布局占位

- 宽度约 80-120px，固定定位或随内容滚动。
- 移动端（<768px）隐藏 Tracing Beam。

## 三、中间：主内容区（分层结构）

整体无背景框、无边框，各层之间以细线分隔（`border-default-200`），保持简洁清晰。

### 3.1 第一层：顶部导航栏

- **返回按钮**：点击返回搜索结果页（`history.back()` 或携带来源参数跳转）。
- **面包屑**：首页 > 搜索 > 文档标题（截断显示）。
- 高度约 48-56px，底部有分隔线。

### 3.2 第二层：文档元信息区

基于 `GET /api/document/docHomeNote/detail/{id}` 响应数据渲染：

| 展示内容 | 数据来源字段                                              | 说明                                 |
| :--- | :-------------------------------------------------- | :--------------------------------- |
| 封面图  | `coverUrl`                                          | 若存在则展示，无封面则不显示                     |
| 标题   | `title`                                             | 主标题，字体最大（text-2xl \~ text-3xl）     |
| 作者   | 来自 `/interaction` 的 `author.name` + `author.avatar` | 与交互信息一同展示                          |
| 发布日期 | `date`                                              | 格式化为 `YYYY年MM月DD日`                 |
| 分类标签 | `category` + `tags`                                 | category 映射为中文分类名；tags 以逗号分隔展示为标签组 |
| 简介   | `description`                                       | 标题下方摘要文字，颜色较浅                      |

**加载态**：使用 HeroUI `Skeleton` 组件展示骨架屏，骨架结构应与实际布局一致。

### 3.3 第三层：交互信息区

基于 `GET /api/document/docHomeNote/interaction/{id}` 响应数据渲染：

| 展示内容 | 数据来源                            | 交互行为                                                       |
| :--- | :------------------------------ | :--------------------------------------------------------- |
| 浏览量  | `viewCount`                     | 仅展示，不可交互（使用 Eye 图标）                                        |
| 点赞   | `likeCount` + `isLiked`         | 点击调用 `POST /like/{id}` 切换状态；已点赞图标实心/高亮，未点赞图标空心             |
| 收藏   | `favoriteCount` + `isFavorited` | 点击调用 `POST /favorite/{id}` 切换状态；已收藏图标实心/高亮                 |
| 作者信息 | `author`                        | 头像 + 昵称 + 粉丝数                                              |
| 关注作者 | `author.isFollowing`            | 点击调用 `POST /follow/{authorId}` 切换状态；已关注显示"已关注"，未关注显示"+ 关注" |

**交互规范**：

- 点赞/收藏/关注操作前校验登录状态，未登录弹出登录引导。
- 操作采用乐观更新（先更新 UI，再等接口返回后纠正）。
- 接口失败时回滚 UI 状态并 toast 提示。
- 点赞数/收藏数/粉丝数使用数字格式化（>999 显示为 1k+）。

**错误处理**：

- 关注自己时后端返回 `"不能关注自己"`，前端 toast 提示并恢复 UI。

### 3.4 第四层：文档正文渲染区

- 使用项目已有的 `@/components/ui/editor/MarkdownPreview` 组件渲染 `content`（Markdown 格式，基于 `react-markdown` + `remark-gfm`）。
- 支持代码高亮、表格、图片等 Markdown 特性。
- 内容区域最大宽度 720-800px，居中显示，保证阅读舒适度。
- 字体大小可调整（小/中/大），默认中等。

**目录锚点**：Markdown 渲染时自动为各级标题（h1/h2/h3/h4）生成 `id`，供右侧目录定位使用。

### 3.5 第五层：评论区（懒加载）

#### 3.5.1 触发机制

- 使用 `IntersectionObserver` 监听评论区占位元素。
- 当占位元素进入视口（rootMargin: 200px）时，发起评论列表请求。
- 加载中显示骨架屏/Spinner。

#### 3.5.2 评论列表

基于 `GET /api/document/docHomeNote/comments/{noteId}` 分页获取：

- **二级评论结构**（B站式）：根评论 → 回复列表。
- **排序切换**：热门（按点赞数降序）/ 最新（按创建时间降序），通过 `sort` 参数控制。
- **分页**：滚动到底部自动加载下一页（`pageNum` + 1），或点击"加载更多"。
- **每条评论展示**：
  - 评论者头像、昵称、粉丝数
  - 评论内容
  - 发布时间（相对时间："刚刚"、"5分钟前"、"2天前"）
  - 点赞数 + 点赞按钮（`POST /comment/like/{commentId}`），支持乐观更新
  - 回复按钮（展开回复输入框）
  - 对于回复，显示 `replyTo` 目标用户昵称（"回复 @张三"）

#### 3.5.3 发表评论

调用 `POST /api/document/docHomeNote/comment`：

- **根评论**：`parentId` 和 `replyToId` 均为 `null`。
- **回复评论**：`parentId` 传根评论ID，`replyToId` 传被回复用户ID。
- 发表成功后新评论插入列表顶部（最新排序）或列表末尾（热门排序）。
- 未登录用户点击评论框时，跳转登录或弹出登录引导。

#### 3.5.4 评论点赞

调用 `POST /api/document/docHomeNote/comment/like/{commentId}`：

- 乐观更新，失败回滚。
- 未登录时引导登录。

### 3.6 分享功能

- **功能描述**：点击分享按钮，将当前文档 URL 复制到剪贴板。
- **交互**：复制成功后 toast 提示"链接已复制到剪贴板"。
- **实现**：使用 `navigator.clipboard.writeText(window.location.href)`。
- **位置**：置于交互信息区（第三层），与点赞/收藏按钮同行。

## 四、右侧：悬浮目录 + 回顶滑块

### 4.1 目录导航

- **位置**：右侧悬浮固定（`position: sticky`，top 偏移约 100px）。
- **数据来源**：从 Markdown 渲染后的 DOM 中提取 h1/h2/h3/h4 标题文本和对应的锚点 `id`。
- **高亮当前**：监听滚动位置（`IntersectionObserver`），高亮用户当前阅读位置对应的目录项。
- **点击跳转**：点击目录项平滑滚动到对应标题位置（`scrollIntoView({ behavior: 'smooth' })`）。
- **样式**：无背景框，文字链接形式；当前项加粗/高亮色；h2 相对 h1 缩进，h3 相对 h2 缩进，h4 再缩进。
- **移动端**：目录折叠为汉堡菜单或底部弹出面板。

### 4.2 回顶滑块（Scroll to Top Lever）

**交互行为**：
- 向下拖动滑块 → 释放后滑块自动向上弹回原位，同时页面平滑滚动到顶部。
- 使用 Framer Motion 的 `drag="y"` + `useSpring` + `onDragEnd` 实现拖拽 + 弹簧回弹动画。
- 拖拽距离与回弹速度正相关（拖得越远，回弹越快）。

**技术方案**：
- 依赖：`motion`（Framer Motion），项目已有。
- `dragConstraints` 限制最大拖拽距离（约 60-80px）。
- `onDragEnd` 时设置 `animate` 回到原点 + `window.scrollTo({ top: 0, behavior: 'smooth' })`。
- 弹簧参数：`type: 'spring', stiffness: 300, damping: 20`，模拟物理回弹手感。

**可见性**：页面滚动超过一屏高度时显示，顶部时隐藏（`opacity` 过渡）。

## 六、技术实现要点

### 6.1 路由设计

```
/front/document/:id    →  文档详情页
```

路由在 `src/router/index.tsx` 中注册，使用 `lazy(import)` 懒加载页面组件。

### 6.2 组件树（建议）

```
DocumentDetailPage
├── TracingBeam                    # 左侧装饰光束
├── DocumentMain                   # 中间主内容区
│   ├── DocumentNavBar             # ① 顶部导航栏
│   ├── DocumentMeta               # ② 元信息区（标题/作者/封面/标签）
│   ├── DocumentInteraction        # ③ 交互信息区（点赞/收藏/关注/分享）
│   ├── MarkdownPreview (content)  # ④ 正文渲染
│   └── CommentSection             # ⑤ 评论区（懒加载）
│       ├── CommentSortBar         # 排序切换（热门/最新）
│       ├── CommentList            # 评论列表（含分页）
│       ├── CommentInput           # 评论输入框
│       └── CommentItem            # 单条评论（含回复）
└── FloatingTOC                    # 右侧悬浮目录 + 回顶滑块
    ├── TOCList                    # 目录列表
    └── ScrollToTopLever           # 回顶拉动条
```

### 6.3 状态管理

- 页面级状态使用 React `useState` / `useReducer`，不存入全局 store。
- 评论列表使用 `useRef` 保存 `IntersectionObserver` 实例。
- 点赞/收藏/关注状态使用乐观更新 + `useCallback`。

### 6.4 移动端适配

- 左侧 Tracing Beam 隐藏。
- 右侧目录改为底部弹出面板或顶部下拉。
- 中间内容区占满宽度，padding 减小。
- 交互按钮触摸区域 ≥ 44px。
- 回顶滑块改为固定右下角圆形按钮（FAB）。

### 6.5 错误与边界处理

| 场景            | 处理方式                        |
| :------------ | :-------------------------- |
| 文档不存在（404）    | 展示"文档未找到"空状态，提供返回按钮         |
| 接口超时/网络错误     | toast 提示"加载失败，请重试"，点击重试按钮   |
| 未登录操作         | toast 提示"请先登录"，弹出登录弹窗或跳转登录页 |
| 评论为空          | 展示"暂无评论，来说点什么吧"引导文案         |
| Markdown 内容为空 | 展示"暂无内容"占位                  |
| 关注自己          | toast 提示"不能关注自己"            |

### 6.6 设计规范

- **无背景框无边框**：各区域不使用 `card` / `bg-content1` 等带背景的容器，内容直接在页面底色上。
- **分隔线**：层级之间使用 `border-b border-default-200` 细线分隔，保证视觉层次。
- **间距**：各层之间间距约 24-32px（`py-6 ~ py-8`）。
- **字体层级**：标题最大，元信息次之，正文标准，辅助信息最小且颜色最浅。
- **主题兼容**：所有颜色使用 HeroUI 语义 token（`text-default-500`、`border-default-200` 等），确保 dark 模式正常切换。
