# 前台-用户作品主页

> **版本**: 2.0
> **更新日期**: 2026-04-30
> **说明**: 基于后端 `DocHomeUser` 接口重新设计，聚焦用户已发布作品的展示与统计。原需求中后端不支持的功能（用户简介、注册时间、等级积分、私信、动态、收藏用户等）已标记为废案。

---

## 废案清单

> 以下功能因后端接口不支持，暂不实现：

| 原需求 | 废案原因 |
| :--- | :--- |
| 用户简介 | 后端 `docHomeUser` 接口不返回用户简介字段 |
| 注册时间 | 后端接口不返回 |
| 最后登录时间 | 后端接口不返回 |
| 用户等级 | 后端接口不返回 |
| 积分/经验值 | 后端接口不返回 |
| 私信用户 | 无对应后端接口 |
| 查看用户动态 | 无对应后端接口 |
| 收藏用户 | 无对应后端接口 |
| 用户发布的工具 | 后端接口仅支持 `note` 和 `video` 两种类型，无 `tool` 类型 |

---

## 依赖接口

| API 路径 | 方法 | 功能 |
| :--- | :--- | :--- |
| `/api/document/docHomeUser/{userId}/works` | GET | 获取用户作品列表（分页，支持按类型筛选） |
| `/api/document/docHomeUser/{userId}/stats` | GET | 获取用户作品统计（总获赞、总浏览、总收藏、笔记数、视频数） |

> **注意**: 用户基本信息（头像、昵称、粉丝数、是否已关注）需从文档/视频详情页的作者信息（`DocHomeAuthor` / `HomeVideoAuthor`）中获取并传递，或后续新增用户信息接口。

## 页面设计

### 路由

- **路径**: `/user/:userId`
- **布局**: `FrontLayout`（前台布局，含顶部导航栏）
- **入口**: 文档详情页 / 视频详情页的作者头像/名称点击跳转

### 整体布局

参考文档详情页（`Document/index.tsx`）的左-中-右三栏布局和视频详情页（`VideoDetail/index.tsx`）的交互模式，采用**左-中-右三栏布局**，整体以**黑白风**为主色调。

```
+----------+-------------------------------------------+----------+
|          |              中间主内容区                   |          |
|  左侧    |  +-------------------------------------+  |  右侧    |
|  装饰    |  |  用户信息卡片区                       |  |  悬浮    |
|  光束    |  |  头像 / 昵称 / 粉丝数 / 关注按钮      |  |  侧边栏  |
|          |  +-------------------------------------+  |          |
|  Tracing |  |  数据统计条                           |  |  作品    |
|  Beam    |  |  总获赞 / 总浏览 / 总收藏 / 笔记数 / 视频数 |  |  筛选    |
|          |  +-------------------------------------+  |  Tab     |
|          |  |  作品筛选栏                           |  |          |
|          |  |  全部 / 笔记 / 视频                   |  |  排序    |
|          |  +-------------------------------------+  |  方式    |
|          |  |  作品网格                             |  |          |
|          |  |  卡片 x N（加载更多）                  |  |          |
|          |  +-------------------------------------+  |          |
+----------+-------------------------------------------+----------+
```

**移动端**: 隐藏左右栏，单栏展示，右侧筛选 Tab 折叠到作品筛选栏中。

### 1. 用户信息卡片区

- **位置**: 中间主内容区顶部
- **数据来源**: 从路由跳转时通过 `location.state` 传递，或后续新增接口
- **展示内容**:

| 字段 | 来源 | 说明 |
| :--- | :--- | :--- |
| 用户头像 | `DocHomeAuthor.avatar` / `HomeVideoAuthor.avatar` | 圆形头像，黑白边框 |
| 用户昵称 | `DocHomeAuthor.name` / `HomeVideoAuthor.name` | 大号加粗，黑色 |
| 粉丝数 | `DocHomeAuthor.fans` / `HomeVideoAuthor.fans` | 数字 + "粉丝" 标签 |
| 关注状态 | `DocHomeAuthor.isFollowing` / `HomeVideoAuthor.isFollowing` | 关注/已关注按钮 |
| 关注按钮 | 调用 `toggleDocHomeNoteFollow` / `toggleHomeVideoFollow` | 黑白风格按钮 |

- **UI 设计**:
  - 水平布局：左侧头像 → 右侧昵称 + 粉丝数 + 关注按钮
  - 头像: `w-16 h-16`，黑白边框 `border-2 border-foreground`
  - 昵称: `text-2xl font-bold text-foreground`
  - 粉丝数: `text-sm text-default-500`
  - 关注按钮: 默认 `bordered` 变体，黑白配色；已关注状态 `flat` 变体，灰色背景
  - 分隔线: 底部 `border-b border-default-200`

### 2. 数据统计条

- **位置**: 用户信息卡片区下方
- **数据来源**: `GET /api/document/docHomeUser/{userId}/stats`
- **展示内容**:

| 统计项 | 字段 | 图标 | 格式化 |
| :--- | :--- | :--- | :--- |
| 总获赞 | `totalLikeCount` | `Heart` (lucide) | >=10000 显示 x.x万 |
| 总浏览 | `totalViewCount` | `Eye` (lucide) | >=10000 显示 x.x万 |
| 总收藏 | `totalFavoriteCount` | `BookMarked` (lucide) | >=10000 显示 x.x万 |
| 笔记数 | `noteCount` | `FileText` (lucide) | 直接显示 |
| 视频数 | `videoCount` | `Video` (lucide) | 直接显示 |

- **UI 设计**:
  - 水平等分排列，5 个统计项一行
  - 每项: 图标 + 数字 + 标签，垂直居中
  - 图标: `w-4 h-4 text-default-400`
  - 数字: `text-lg font-bold text-foreground`
  - 标签: `text-xs text-default-500`
  - 底部分隔线: `border-b border-default-200`
  - 加载态: 使用 `animate-pulse` 骨架屏占位

### 3. 作品筛选栏

- **位置**: 数据统计条下方
- **功能**: 按类型筛选用户作品
- **筛选选项**:

| Tab 标签 | type 参数 | 说明 |
| :--- | :--- | :--- |
| 全部 | 不传 | 查询笔记+视频，合并排序 |
| 笔记 | `note` | 仅查询笔记 |
| 视频 | `video` | 仅查询视频 |

- **UI 设计**:
  - 使用 HeroUI `Tabs` 组件，黑白风格
  - 选中态: `text-foreground` + 底部黑色指示条
  - 未选中态: `text-default-400`
  - 右侧显示作品总数: `text-sm text-default-500`
  - 移动端: 右侧筛选 Tab 折叠到此栏

---

### 4. 作品网格

- **位置**: 作品筛选栏下方
- **数据来源**: `GET /api/document/docHomeUser/{userId}/works`
- **展示内容**:

每张作品卡片展示：

| 字段 | 说明 |
| :--- | :--- |
| `coverUrl` | 封面图，4:3 比例，圆角 |
| `title` | 标题，单行截断 |
| `type` | 类型角标（笔记/视频），左上角 |
| `category` | 分类标签 |
| `viewCount` | 浏览量 + Eye 图标 |
| `likeCount` | 点赞数 + Heart 图标 |
| `favoriteCount` | 收藏数 + BookMarked 图标 |
| `createTime` | 创建时间，格式化为月/日 |

- **UI 设计**:
  - 网格布局: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6`
  - 卡片风格: 参考现有 `WorkCard` 组件，黑白配色
  - 封面区: 黑白滤镜 hover 效果（`grayscale group-hover:grayscale-0`），hover 时封面微放大
  - 类型角标: 左上角小标签，笔记 `bg-foreground text-background`，视频 `bg-default-500 text-background`
  - 无封面时: 显示类型文字占位，`bg-default-100`
  - 底部信息: 黑色文字 + 灰色图标，日期右对齐
  - 点击跳转: 笔记 → `/document/:id`，视频 → `/video/:id`

- **分页方式**: 加载更多按钮（参考 `WorksGrid` 组件）
  - 按钮: `rounded-full`，黑白风格
  - 加载中: 显示 Spinner
  - 无更多数据: 隐藏按钮

- **空状态**: 使用 `StatusState type="empty"`
- **加载态**: 使用 `WorkSkeleton` 骨架屏，6 个占位卡片

## 状态管理

### 页面状态

| 状态 | 处理方式 |
| :--- | :--- |
| 加载中 | 骨架屏（用户信息区 + 统计条 + 作品网格） |
| 加载失败 | `StatusState type="error"` + 返回搜索页链接 |
| 用户无作品 | `StatusState type="empty"` + "该用户暂无作品" |
| 加载更多失败 | toast 提示，不清除已有数据 |

### 关注操作

- 使用乐观更新策略（参考 `useDocumentDetail` Hook）
- 关注/取关调用 `toggleDocHomeNoteFollow` 或 `toggleHomeVideoFollow`
- 操作中按钮显示 Spinner，禁用点击
- 失败时回滚状态并 toast 提示

## 自定义 Hook

### `useUserHome(userId: string)`

封装用户主页数据获取与交互逻辑：

```typescript
interface UseUserHomeReturn {
  // 作品列表
  works: DocHomeUserWorksVo[]
  worksLoading: boolean
  worksError: string | null
  // 统计数据
  stats: DocHomeUserStatsVo | null
  statsLoading: boolean
  // 分页
  pageNum: number
  hasMore: boolean
  loadMore: () => void
  // 筛选
  typeFilter: 'all' | 'note' | 'video'
  setTypeFilter: (type: 'all' | 'note' | 'video') => void
  // 关注
  isFollowing: boolean
  followLoading: boolean
  handleFollow: () => void
}
```

**数据加载策略**:
1. 进入页面 → 并行请求 `works`（首页）+ `stats`
2. 切换筛选 Tab → 重置页码，重新请求 `works`
3. 加载更多 → 追加请求下一页 `works`
4. 关注操作 → 乐观更新 `isFollowing`，失败回滚
