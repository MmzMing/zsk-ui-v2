# 路由权限管理

## 一、技术选型

| 技术 | 版本 | 用途 |
|------|------|------|
| React Router DOM | ^7.12.0 | 路由管理 |
| Zustand | ^5.0.9 | 状态管理（存储用户权限） |
| TypeScript | ~5.9.3 | 类型安全 |

## 二、权限控制流程

用户访问路由 → 检查登录状态 → 未登录重定向登录页 → 已登录检查路由权限配置 → 有权限配置检查用户权限列表 → 权限匹配允许访问 → 不匹配显示无权限页面

## 三、权限判断机制

采用三层校验机制：

1. **角色校验**：检查用户角色是否在允许的角色列表中
2. **权限校验**：检查用户权限列表是否包含所需权限
3. **超级管理员豁免**：super_admin角色自动拥有所有权限

## 四、权限标识规范

采用 `模块:子模块:操作` 格式：

- **模块**：业务领域（system、ops、robot、video、document、personnel）
- **子模块**：具体功能（config、monitor、qq、wechat、dingding、list）
- **操作**：动作类型（view、edit、delete、audit、upload）

**示例**：
- system:config:view - 系统配置查看权限
- ops:monitor:view - 运维监控查看权限
- robot:qq:view - QQ机器人查看权限
- video:upload:view - 视频上传查看权限
- document:audit:view - 文档审核查看权限

## 五、用户角色定义

### 角色类型

| 角色 | 标识 | 权限范围 |
|------|------|----------|
| 超级管理员 | super_admin | 拥有所有权限，无需配置 |
| 管理员 | admin | 拥有管理类权限，可管理用户/角色/菜单 |
| 普通用户 | user | 拥有基础访问权限 |
| 访客 | guest | 仅拥有公开访问权限 |

### 角色权限差异

**super_admin（超级管理员）**：
- 自动拥有所有权限，不受权限配置限制
- 可访问所有管理页面
- 可执行所有操作

**admin（管理员）**：
- 需要配置具体权限
- 可管理用户、角色、菜单
- 受权限配置限制

**user（普通用户）**：
- 仅拥有基础业务权限
- 不可访问管理页面

**guest（访客）**：
- 仅能访问公开页面
- 无后台访问权限

## 六、路由权限配置

### 配置结构

路由权限配置包含以下属性：
- **roles**：允许访问的角色列表
- **permissions**：允许访问的权限列表（满足任一即可）
- **requireAuth**：是否需要登录
- **redirectTo**：无权限时重定向路径

### 权限配置策略

**页面级权限**：
- 需要登录但无特定权限要求的页面（如仪表盘）：仅配置 `requireAuth: true`
- 需要特定角色的页面（如用户管理）：配置 `roles` 数组
- 需要特定权限的页面（如系统配置）：配置 `permissions` 数组

**功能级权限**：
- 在组件内部使用权限Hook判断具体操作权限
- 支持多权限组合判断（全部满足/任一满足）

## 七、核心组件

### 权限路由守卫（PermissionRoute）

用于保护需要特定权限的路由，支持：
- 角色权限检查
- 功能权限检查
- 自定义无权限回退组件

### 后台布局守卫（AdminLayoutGuard）

自动检查当前路由的权限配置，统一处理登录状态和权限验证。

### 权限判断Hook（usePermission）

提供权限检查功能：
- hasRole：检查角色权限
- hasPermission：检查单个权限
- hasAllPermissions：检查所有权限（全部满足）
- hasAnyPermission：检查任一权限（满足其一）
- filterMenusByPermission：过滤有权限的菜单

## 八、权限缓存策略

- 用户权限信息存储在Zustand Store中
- 使用persist中间件持久化到localStorage
- 登录时从后端获取最新权限并更新缓存
- Token过期时自动清除权限缓存

## 九、最佳实践

### 权限粒度设计

- **粗粒度权限**：用于页面级访问控制（如 video:view）
- **细粒度权限**：用于功能级操作控制（如 video:upload、video:audit）
- **角色权限**：用于管理类功能（如 admin、super_admin）

### 权限命名规范

保持统一的命名格式：`模块:子模块:操作`

### 性能优化

- 使用useMemo缓存权限计算结果
- 使用useCallback缓存权限判断函数
- 菜单权限过滤使用useMemo避免重复计算

## 十、权限配置示例

### 路由权限映射

| 路由路径 | requireAuth | roles | permissions |
|----------|-------------|-------|-------------|
| /admin/dashboard | true | - | - |
| /admin/system/config | true | - | system:config:view |
| /admin/ops/monitor | true | - | ops:monitor:view |
| /admin/robot/qq | true | - | robot:qq:view |
| /admin/video/list | true | - | video:list:view |
| /admin/personnel/user | true | admin, super_admin | - |
| /admin/personnel/role | true | admin, super_admin | - |
| /admin/personnel/menu | true | admin, super_admin | - |

### 角色默认权限

**super_admin**：
- 拥有所有权限（通配符 *）

**admin**：
- system:config:view
- ops:monitor:view
- robot:qq:view
- robot:dingding:view
- robot:wechat:view
- robot:napcat:view
- video:list:view
- video:upload:view
- video:audit:view
- document:list:view
- document:edit:view
- document:upload:view
- document:audit:view
- personnel:user:view
- personnel:role:view
- personnel:menu:view

**user**：
- video:list:view
- document:list:view

**guest**：
- 无权限
