# 字典缓存管理

## 一、概述

字典缓存管理系统基于 **Zustand + 版本控制** 实现，提供自动版本检测与增量更新功能。通过版本号比对机制，避免不必要的全量拉取，减少网络请求与渲染开销。

### 核心特性

- **版本控制**：每个字典类型独立维护版本号，支持精准增量更新
- **自动检测**：组件挂载时自动执行版本比对，无需手动刷新
- **批量优化**：应用启动时按版本差异按需拉取，避免全量加载
- **内存缓存**：字典数据存储在 Zustand Store 中，页面级共享
- **类型安全**：完整的 TypeScript 类型定义

### 3.2 Zustand Store

**文件位置**：`src/stores/dict.ts`

```typescript
interface DictStoreState {
  cache: Record<string, DictCacheItem>  // 内存缓存
  
  getDictData: (dictType: string) => SysDictDataCache[]  // 获取字典数据
  getVersion: (dictType: string) => number                // 获取版本号
  setCache: (dictType: string, cacheVO: DictCacheVO) => void  // 设置缓存
  setBatchCache: (allCache: Record<string, DictCacheVO>) => void  // 批量设置
  removeCache: (dictType: string) => void                 // 清除指定缓存
  clearAllCache: () => void                               // 清除所有缓存
}
```

**核心方法说明**：

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `getDictData` | `dictType: string` | `SysDictDataCache[]` | 获取字典数据，按 `dictSort` 排序 |
| `getVersion` | `dictType: string` | `number` | 获取本地缓存版本号，不存在返回 0 |
| `setCache` | `dictType`, `cacheVO` | `void` | 更新单个字典类型的缓存 |
| `setBatchCache` | `allCache` | `void` | 批量更新多个字典类型的缓存 |
| `removeCache` | `dictType` | `void` | 清除指定字典类型的缓存 |
| `clearAllCache` | - | `void` | 清除所有字典缓存 |

### 3.3 核心 Hook

**文件位置**：`src/hooks/useDict.ts`

#### 3.3.1 `useDict` - 单个字典获取

```typescript
function useDict(dictType: string): UseDictReturn

interface UseDictReturn {
  data: SysDictDataCache[]      // 字典数据列表
  loading: boolean              // 加载状态
  refresh: () => Promise<void>  // 手动刷新方法
  clearCache: () => void        // 清除缓存方法
}
```

**工作流程**：

1. 组件挂载时，`useEffect` 自动触发 `refresh()`
2. `refresh()` 调用 `getDictCacheVersion(dictType)` 获取服务端版本
3. 调用 `getVersion(dictType)` 获取本地版本
4. 比对版本：`serverVersion > localVersion` 时拉取最新数据
5. 调用 `getDictCacheByTag(dictType)` 获取最新数据
6. 调用 `setCache(dictType, cacheVO)` 更新缓存
7. 组件重新渲染，显示最新数据

**使用示例**：

```tsx
// 在表单中使用
const { data, loading } = useDict('sys_user_sex')

<Select label="性别" isLoading={loading}>
  {data.map(item => (
    <SelectOption key={item.dictValue} value={item.dictValue}>
      {item.dictLabel}
    </SelectOption>
  ))}
</Select>

// 手动刷新（如字典编辑后）
const { refresh } = useDict('sys_user_sex')
await refresh()  // 触发版本比对并更新
```

#### 3.3.2 `useDictAll` - 批量加载所有字典

```typescript
function useDictAll(): { loadAll: () => Promise<void>, loading: boolean }
```

**工作流程**：

1. 调用 `getDictCacheTags()` 获取服务端全部字典类型标签列表
2. 遍历每个标签，调用 `getDictCacheVersion(tag)` 获取服务端版本
3. 调用 `getVersion(tag)` 获取本地版本
4. 收集版本不一致的字典类型到 `needUpdate` 数组
5. 使用 `Promise.all` 并发拉取需要更新的字典数据
6. 调用 `setBatchCache(allCache)` 批量更新缓存

**性能优势**：

| 场景 | 修改前 | 修改后 |
|------|--------|--------|
| 全量加载 | 一次请求获取全部字典数据 | 先查版本标签，仅拉取变化的 |
| 网络开销 | 传输全部字典数据 | 仅传输变更部分 |
| 响应速度 | 等待全部数据加载完成 | 并发拉取，增量更新 |

**使用示例**：

```typescript
// 在应用初始化时调用（useUserInit.ts）
const { loadAll: loadAllDicts } = useDictAll()

useEffect(() => {
  loadAllDicts()  // 应用启动时批量加载字典
}, [])
```

## 六、技术要点

### 6.1 版本比对机制

```typescript
// 核心比对逻辑
const serverVersion = await getDictCacheVersion(dictType)  // 服务端版本号
const localVersion = getVersion(dictType)                   // 本地缓存版本号

if (serverVersion > localVersion) {
  // 版本不一致，拉取最新数据
  const cacheVO = await getDictCacheByTag(dictType)
  setCache(dictType, cacheVO)
}
```

**版本号设计**：
- 版本号使用时间戳格式（如 `1717234567890`）
- 每次字典数据变更，后端递增版本号
- 前端通过比较版本号判断是否需要更新

### 6.2 并发优化

批量加载时使用 `Promise.all` 并发请求：

```typescript
const updatePromises = needUpdate.map(async (tag) => {
  const cacheVO = await getDictCacheByTag(tag)
  return { tag, cacheVO }
})

const results = await Promise.all(updatePromises)
```

**优势**：
- 减少总耗时，所有请求并行发起
- 避免串行请求的等待时间
- 提高应用启动速度

### 6.3 防重复初始化

使用 `useRef` 防止组件重复挂载时重复请求：

```typescript
const initializedRef = useRef(false)

useEffect(() => {
  if (!initializedRef.current) {
    initializedRef.current = true
    refresh()
  }
}, [refresh])
```

### 6.4 内存缓存设计

## 七、注意事项

### 7.1 缓存生命周期

- 字典缓存存储在 **Zustand Store 内存** 中
- 页面刷新后缓存丢失，需要重新加载
- 应用启动时通过 `loadAllDicts()` 自动加载

### 7.2 字典编辑后更新

字典数据在后端修改后，前端需要更新缓存：

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 页面刷新 | 重新走初始化流程 | 简单场景 |
| 手动 refresh | 调用 `refresh()` 方法 | 精确控制 |
| 清除缓存 | 调用 `clearCache()` 后重新加载 | 强制刷新 |

### 7.3 性能考虑

- 字典数据量较小时，版本比对开销可忽略
- 字典数据量大时，增量更新优势明显
- `Promise.all` 并发请求数量受浏览器限制（通常 6 个）
- 大量字典类型时建议后端优化版本查询接口

### 7.4 错误处理

版本比对失败时不阻断渲染：

```typescript
try {
  const serverVersion = await getDictCacheVersion(dictType)
  // ...
} catch (error) {
  console.error(`字典缓存刷新失败 [${dictType}]：`, error)
} finally {
  setLoading(false)
}
```

- 失败时保留现有缓存数据
- 仅记录错误日志，不影响页面展示
- 下次组件挂载或手动刷新时重试
